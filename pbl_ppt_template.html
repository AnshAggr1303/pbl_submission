The Problem

Small businesses that hold physical stock — restaurants, pharmacies, kirana stores — are stuck in the same loop. Manual counting. Wasted stock. No visibility. No alerts.

  • —Manual counting every night with a notebook
  • —Over-ordering paneer weekly because nobody tracks history
  • —Expired stock thrown away — no expiry alerts
  • —Runs out mid-service — no forecasting, no reorder automation
  • —Existing tools cost ₹2000+/month and require an accountant
Our approach
Universal Platform — Any Industry
Same codebase. Industry selection during onboarding configures labels, templates, and unit defaults. Restaurant calls it "Recipes". Pharmacy calls it "Formulas". Manufacturing calls it "Bill of Materials".
Cost model
BYOK — Bring Your Own Keys
Users bring their own free Groq and Gemini API keys. Platform running cost = near zero. Keys encrypted with AES-256-GCM before storage. Never visible in browser.

Data Layer
Supabase
Postgres (11 tables) Row Level Security Postgres Triggers Supabase Auth Supabase Storage Atomic RPCs
Rule D1
Stock as a Ledger
current_stock is never updated directly. Every change inserts a row into transactions. A Postgres trigger computes the new balance atomically. Full audit trail. No race conditions. No corruption.
Rule D2
Atomic Multi-table Writes
Bill confirmation, recipe deduction, and item creation all use Postgres RPC functions. Either everything commits or nothing does. Partial saves are impossible by design.
Agent design
Human in the Loop
All three agents write to agent_actions with status: pending. Nothing executes without explicit user approval. Agents propose. Humans decide.

What It Does

📷
Tesseract.js + Gemini 2.0 Flash
OCR Bill Scanner
Photograph any supplier bill. Client-side Tesseract pre-processes for speed. Gemini Vision extracts items, quantities, units, and prices server-side. Smart token parser handles Indian bill formats across 3 patterns.
// Sample bill → 14 items, one click
Paneer Fresh → 5 kg @ ₹340
Maida → 10 kg @ ₹38
Mustard Oil → 5 litre @ ₹180
→ all added to inventory instantly
📖
Atomic Postgres RPC + Trigger
Recipe Engine
Define recipes once. Sell a dish — all ingredients deduct automatically. Sufficiency check runs server-side before any write. Stock never goes below zero. Low stock triggers agent actions.
// Sell 3× Veg Roll
Paneer: -300g  340g → 40g
Maida:  -150g  680g → 530g
Mayo:   -6g    120g → 114g
⚠ Paneer below reorder point
🤖
Groq LLM + Vercel Cron
3 Autonomous AI Agents
Independent specialist agents sharing state via Supabase. No direct agent-to-agent communication. Every action requires human approval — agents propose, humans decide.
Reorder
Drafts WhatsApp supplier message when item hits threshold. 8AM daily + event-triggered.
Shrinkage
Weekly variance — expected vs actual. Flags theft or spillage in ₹. Monday 7AM.
Forecast
30-day pattern analysis. Detects weekend demand spikes. 6AM daily briefing.
💬
Gemini Flash + Fallbacks
Co-pilot Chat
Ask anything about inventory in plain English. Rule-based fallbacks work even without API keys.
you → how much paneer left?

ai  → You have 40g of Paneer.
